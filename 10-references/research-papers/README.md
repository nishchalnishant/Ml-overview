## The Papers Every ML Interview Cares About

**You should be able to describe these in 2-3 sentences each:**

1. **Attention Is All You Need** (Vaswani et al., 2017) — Transformer architecture, scaled dot-product attention, multi-head attention
2. **BERT** (Devlin et al., 2018) — Masked language modeling, bidirectional pretraining, fine-tuning paradigm
3. **GPT-3** (Brown et al., 2020) — Scale unlocks in-context learning without fine-tuning
4. **ResNet** (He et al., 2015) — Skip connections solve vanishing gradients
5. **XGBoost** (Chen & Guestrin, 2016) — Regularized boosting, second-order gradients
6. **Hidden Technical Debt in ML** (Sculley et al., 2015) — CACE principle, why ML systems accumulate debt faster than software
7. **PagedAttention / vLLM** (Kwon et al., 2023) — KV cache management for LLM serving
8. **LoRA** (Hu et al., 2021) — Low-rank adaptation for efficient fine-tuning
9. **RLHF** (Ouyang et al., InstructGPT, 2022) — Aligning LLMs with human preferences
10. **CLIP** (Radford et al., 2021) — Vision-language alignment via contrastive pretraining

## How to Use These in Interviews

When a paper is relevant, mention it like this:

> "This is the approach from the [paper name] — they showed that [key finding], which is why I'd [approach]."

You don't need the exact citation. The idea and attribution are enough.
